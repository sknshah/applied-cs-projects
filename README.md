# Applied CS Projects

Small systems/algorithms projects (game design, data structures, parsers, etc.) built independently, from scratch — some inspired by concepts encountered in coursework, but not derived from any graded assignment's solution.

**Suggested GitHub topics:** `java` `algorithms` `software-design` `game-development`

## Why this exists, and what it isn't

This repo exists to showcase systems-design and algorithms skills without any academic-integrity ambiguity. That means:

- Nothing here is a reworded or "restyled" version of a graded solution — Penn's (and most CS departments') integrity policies bar posting assignment solutions publicly regardless of how much the code is changed, since courses reuse assignments across semesters.
- Each project is labeled with what class concept it's *inspired by* (e.g. "inspired by a strategy-game assignment from an intro programming course") without naming the specific course/assignment, and is built with an original spec, original class design, and original test suite.
- If you'd rather include an *actual* past assignment here, get written permission from the current instructor first — then it can go in a clearly separate, permission-noted subfolder.
